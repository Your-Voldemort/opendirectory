# skills/

**Purpose:** Registry of AI agent skills—modular, discoverable integrations for the CLI. Each skill is independently developed and installable.

---

## STRUCTURE

```
skills/
├── blog-cover-image-cli/           # Google Gemini + Sharp image gen CLI
├── twitter-GTM-find-skill/         # Twitter scraper + LLM analysis (Apify)
├── yc-intent-radar-skill/          # Y Combinator jobs scraper (Playwright)
├── hackernews-intel/               # Hacker News monitoring (SQLite)
├── newsletter-digest/              # RSS parser + digest generation
├── docs-from-code/                 # Code-to-docs extraction (ts-morph)
├── cold-email-verifier/            # Python skill for email validation
└── registry.json                   # Auto-generated skill registry
```

---

## SKILL TEMPLATE

Each skill follows this pattern:

```
skill-name/
├── package.json           # Skill metadata, deps, scripts
├── src/ or scripts/       # Source code (TS/JS/Python)
├── README.md              # Skill docs
└── [optional] tsconfig.json, tests/
```

**Key:** Skills are **independent packages** managed by pnpm workspace. Each has its own `package.json` and dependencies.

---

## ADDING A NEW SKILL

1. **Create directory** in `skills/my-new-skill/`
2. **Add `package.json`** with name, description, version, scripts
3. **Implement logic** in `src/` or `scripts/`
4. **Run root build:**
   ```bash
   pnpm install
   pnpm exec tsx scripts/build-registry.ts
   ```
5. **Registry auto-updates** → `registry.json` reflects new skill
6. **CLI copies on build** → `packages/cli/skills/` includes your skill

---

## UNIQUE PATTERNS

1. **Registry Discovery:** `build-registry.ts` scans all `skills/*/package.json` → generates centralized `registry.json`
2. **Workspace Sync:** pnpm workspaces handles all deps across skills; no manual npm installs needed
3. **Python Integration:** `cold-email-verifier` uses `requirements.txt` (separate from pnpm)—install manually or via CI
4. **Duplication Post-Build:** Skills mirrored to `packages/cli/skills/` during CLI build; source of truth is here
5. **Independent Versioning:** Each skill can have its own version; CLI version bumps separately

---

## KEY FILES

| File | Purpose |
|------|---------|
| `package.json` | Workspace definition, lists `skills/**` as packages |
| `registry.json` (auto) | Discoverable skill registry, generated by build script |
| `scripts/build-registry.ts` | Scans skills/, generates registry.json |

---

## CONVENTIONS (SKILLS-SPECIFIC)

1. **Naming:** Kebab-case skill directories (e.g., `twitter-GTM-find-skill`)
2. **Package name:** Often matches directory or uses scoped name
3. **No global CLI:** Each skill is a module/library, not a global command (except when standalone like blog-cover-image-cli)
4. **Dependencies:** Added to skill's `package.json`; pnpm workspace resolves
